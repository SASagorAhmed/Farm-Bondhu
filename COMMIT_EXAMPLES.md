# Commit Message Examples

Use these as ready-to-use templates. Keep commits small and logical.

1. `feat(auth): add OTP verification flow for registration`
2. `fix(auth): handle invalid token response on session restore`
3. `feat(admin): add pending approvals summary cards`
4. `fix(admin): prevent duplicate action on approval buttons`
5. `feat(marketplace): add seller-wise product filtering`
6. `fix(marketplace): correct order total calculation for mixed carts`
7. `feat(community): add category filter chips on feed page`
8. `fix(community): preserve scroll position after post details view`
9. `feat(dashboard): add mortality trend chart for last 30 days`
10. `fix(dashboard): correct date range labels in finance widgets`
11. `feat(vet): add consultation status badges in waiting room`
12. `fix(vet): resolve prescription save failure on empty notes`
13. `feat(learning): add searchable guide list with topic tags`
14. `fix(profile): update role request form validation messages`
15. `feat(api): add v1 route for notifications unread count`
16. `fix(api): return consistent error shape for validation failures`
17. `refactor(api-client): centralize API base URL and auth headers`
18. `chore(env): document Vercel and local environment variables`
19. `docs(deploy): add split frontend-backend Vercel deployment guide`
20. `test(frontend): add auth provider session bootstrap test`

## Quick Format

`type(scope): short summary`

Common types:
- `feat`
- `fix`
- `refactor`
- `docs`
- `test`
- `chore`
